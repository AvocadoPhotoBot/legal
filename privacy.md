## 2. Какие данные мы обрабатываем

При взаимодействии с ботом Оператор получает и хранит следующие данные:

### 2.1. Данные из Telegram
- **Telegram user ID** — уникальный числовой идентификатор аккаунта
- **Имя** — имя, указанное в профиле Telegram
- **Username** — псевдоним (@username), если установлен

Эти данные передаются автоматически платформой Telegram при первом запуске бота и необходимы для стабильной работы сервиса, а также для диагностики и устранения технических проблем при их возникновении.

### 2.2. Данные о использовании сервиса
- Баланс кредитов (оплаченных и бесплатных)
- Общее количество выполненных генераций
- Дата регистрации в боте
- Статус подписки на Telegram-канал сервиса
- Реферальные связи (кто пригласил пользователя)

### 2.3. Платёжные данные
- Идентификатор платежа в системе Telegram
- Приобретённый пакет и сумма платежа
- Статус и дата транзакции

Данные банковской карты Оператор **не получает и не хранит** — они обрабатываются исключительно инфраструктурой платёжного провайдера.

### 2.4. Фотографии пользователей
Загружаемые фотографии передаются стороннему AI-сервису для генерации изображений и **не сохраняются** на серверах Оператора. В базе данных хранятся только технические идентификаторы файлов Telegram (file_id) — ссылки на файлы в инфраструктуре Telegram.

**Временное хранение результата пробной генерации.** При использовании бесплатного кредита чистое (без водяного знака) сгенерированное изображение временно размещается в защищённом облачном хранилище Supabase Storage на срок не более 24 часов. Изображение удаляется автоматически: немедленно после оплаты и доставки Заказчику либо по истечении 24 часов, если оплата не поступила. Оператор не просматривает и не анализирует эти изображения.

## 3. Цели обработки данных

Оператор обрабатывает данные исключительно в следующих целях:

| Цель | Данные |
|---|---|
| Идентификация пользователя в системе | Telegram ID, имя, username |
| Учёт баланса и списание кредитов | Баланс кредитов, история генераций |
| Обработка платежей и зачисление кредитов | Платёжные данные |
| Работа реферальной программы | Реферальные связи |
| Техническая поддержка | Все перечисленные данные |
| Информирование об акциях и обновлениях | Telegram ID |

Оператор **не продаёт** и **не передаёт** данные пользователей третьим лицам в маркетинговых целях.

---

## 4. Хранение данных

4.1. Данные хранятся в защищённой облачной инфраструктуре с использованием базы данных **Supabase** (PostgreSQL). Провайдер обеспечивает безопасность инфраструктуры: шифрование при передаче (TLS) и хранении данных, контроль доступа и мониторинг.

4.2. Доступ к базе данных имеет только Оператор.

4.3. Данные хранятся в течение всего срока использования сервиса пользователем. При удалении аккаунта (по запросу) данные удаляются в течение 30 дней.

---

## 5. Передача данных третьим лицам

Для оказания услуги Оператор использует следующих сервис-провайдеров, которым могут передаваться данные в минимально необходимом объёме:

| Провайдер | Назначение | Передаваемые данные |
|---|---|---|
| **Telegram** | Платформа бота и платежи | Данные профиля (стандартный механизм Telegram API) |
| **Supabase** | Облачная база данных и хранилище | Все хранимые данные (см. раздел 2); временно — чистое изображение пробной генерации (до 24 ч) |
| **FAL.ai** | Генерация AI-изображений | Загруженная пользователем фотография (не сохраняется) |

Все провайдеры работают в соответствии с их собственными политиками конфиденциальности и применимым законодательством.
